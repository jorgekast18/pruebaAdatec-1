Descripción de la solución:

Pokedex es un sistema que permite consultar los Pokemons que se encuentran en el sistema por medio de un WebService,además se crearon WebServices diferentes para las tareas de Crear, Modificar y Eliminar un Pokemon.

Archivos creados:

En AngularJs se crearon 4 archvios, 2 Servicios y 2 controladores, además de las respectivas vistas.

  1. services: createPokemon.js
  2. services: viewPokemon.js
  3. controllers: createPokemon.js
  4. controllers: viewPokemon.js

En Django se creó el ambiente virtual "python", se migró la base de datos "db.sqlite3" y se creó el usuario "admin", seguidamente se procede a configurar los diferentes archivos de python para que respondan a las peticiones que se hagan por medio de AngularJs.

El primer archivo que se configró fue "admin.py" para que se mostrara en el administrador la tabla (Pokemon) qe se creó en el modelo

Seguidamente se configuró el archivo "urls.py", para cuando llegue una petición reconozca los datos que están en la url y se dirija al archivo "views.py", en el cual se configuraron 2 funciones ("poke_list" y "poke_detail") las cuales identifican las peticiones que se reciben ("POST", "GET", "PUT", "OPTION", "DELETE") y de esa forma retorna los datos de acuerdo a la petición.
